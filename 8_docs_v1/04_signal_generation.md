## Overview

Section 4 builds the complete ML signal pipeline. It takes the features produced in Section 3 and produces a calibrated BUY/SELL/WAIT signal with a confidence score, probability estimates, and a regime drift warning. The pipeline replaces the TradingAgents debate chain entirely and runs with no GPU requirement and no API calls during training.

---

## Architecture

```
feature_builder.py
    --> rc_temporal.py          [Echo State Network, temporal patterns]
    --> kalman_risk.py          [dynamic stop-loss features]
    --> advanced_price_features [RWI, OU, QV, D/D, HJB]
    --> xgboost_model.py        [train / predict]
    --> signal_engine.py        [orchestrator, sentiment overlay]
```

## Files

### `4_signals/feature_builder.py`

Builds the feature matrix for a single ticker over a historical period. Called by `build_multi_ticker_dataset()` in `xgboost_model.py` for training, and by `predict_signal()` for inference.

**`fetch_ohlcv()`** downloads OHLCV data from yfinance with a configurable lookback period. Handles MultiIndex column flattening and drops rows with any NaN values.

**`add_technical_features()`** computes 20 technical features via pandas-ta:

- Trend: SMA(20), SMA(50), EMA(10), EMA(20)
- Momentum: RSI(14), RSI(7), MOM(10), MACD, MACD signal, MACD histogram
- Volatility: ATR(14), ATR(7), Bollinger Bands (width, percent position)
- Volume: VWMA(20), volume ratio vs 20-day average
- Price-derived: 1/5/10/20-day returns, high-low ratio, gap
- Binary flags: above SMA(20), above SMA(50), golden cross

**`add_fundamental_features()`** pulls static fundamental data from `yfinance.Ticker.info`: P/E, forward P/E, price-to-book, debt/equity, ROE, beta, short ratio, distance from 52-week high and low.

Note: fundamental features are static per query date. They do not change row by row within a training dataset. SHAP analysis showed near-zero importance for these features after adding advanced price features. They remain in the pipeline for completeness but may be removed in a future revision.

**`add_labels()`** creates the target variable for supervised learning. Uses Triple Barrier Labeling (see `xgboost_model.py` for full description).

**`get_feature_columns()`** returns the canonical list of feature column names used as the XGBoost input. This list must be consistent between training and inference.

### `4_signals/rc_temporal.py`

Implements an Echo State Network (ESN), a form of Reservoir Computing.

**Why ESN instead of LSTM.**

LSTM requires backpropagation through time, is slow to train on CPU, and tends to overfit on small financial datasets. ESN fixes the reservoir (a randomly initialised recurrent network) and only trains the linear output layer (ridge regression). This means training time is measured in seconds rather than minutes, and the model generalises better with limited data.

**Architecture:**

| Parameter | Value | Notes |
|---|---|---|
| reservoir_size | 200 | Number of recurrent units |
| spectral_radius | 0.95 | Controls memory decay. Values close to 1.0 retain longer history. |
| sparsity | 0.1 | 10% of reservoir connections are active |
| input_scaling | 0.5 | Scales input weights |
| leak_rate | 0.3 | Controls update speed. Lower = more memory of past states. |

**Z-score normalisation.** Raw ESN decision values are not comparable across different tickers because each ESN is trained on its own price scale. A rolling z-score normalisation (window=100, min_periods=20) is applied after signal generation. After normalisation, a z-score of 2.0 consistently means a two-sigma signal regardless of whether it came from TSLA or SPY.

**Leakage prevention.** The ESN is trained on the first 70% of each ticker's data. Signals for the remaining 30% are generated by calling `predict()` on incrementally growing price windows, never using future data.

The 1-bar shift applied after signal generation (`df["esn_signal"] = df["esn_signal"].shift(1)`) ensures the signal from bar T is not used as a feature for bar T's label. Without this shift, the ESN has access to information from the same bar it is trying to predict.

**`train_esn()`** trains an ESN on a price series and optional label series and returns the fitted model with accuracy.

**`get_esn_signal()`** loads or trains an ESN and returns the latest prediction.

**`EchoStateNetwork.save()` / `.load()`** serialises the fitted model to disk via joblib. The model file is saved to `4_signals/esn_model.pkl`.

### `4_signals/xgboost_model.py`

The core signal model. Trains a gradient-boosted classifier on multi-ticker data and produces calibrated buy/sell probabilities.

**Triple Barrier Labeling**

Standard labels (price up in 5 days = BUY, else SELL) are noisy because a 0.01% gain counts the same as a 5% gain. Triple Barrier Labeling assigns labels based on which barrier the price path hits first.

| Barrier | Level | Label |
|---|---|---|
| Upper (profit target) | Entry price + 1.1 × ATR(14) | 1 (BUY) |
| Lower (stop-loss) | Entry price - 1.0 × ATR(14) | 0 (SELL) |
| Vertical (time limit) | After 5 trading days | 0 (SELL) if neither hit |

The profit target was set to 1.1 × ATR (slightly asymmetric) after the symmetric 1.0/1.0 configuration produced a 50/50 BUY/SELL split that was insufficiently informative, and the original 1.5/1.0 configuration produced a 25/75 split that caused the model to default to SELL. The 1.1/1.0 setting produces approximately a 37% BUY ratio which is class-balanced enough for training.

Additional metadata saved per row: `tb_exit_days` (actual holding period before a barrier was hit) and `tb_exit_type` (profit, stop, or time).

**Multi-Ticker Dataset**

Training data is built from 10 tickers: AAPL, MSFT, GOOGL, AMZN, NVDA, META, JPM, SPY, QQQ, TSLA. Each ticker contributes approximately 480 rows (2 years of daily data), producing approximately 4,800 total training samples before filtering.

The global model learns patterns that generalise across different asset types, sectors, and volatility regimes. A model trained on only one ticker tends to overfit its specific characteristics.

**Feature Set (CORE_FEATURES)**

19 features are used for training after removing features with near-zero SHAP importance:

```
Technical:    atr_14, macd, macd_hist, bb_width, bb_pct,
              return_20d, return_5d, return_1d, vol_ratio,
              rsi_14, rsi_7, golden_cross
Kalman:       kalman_deviation, kalman_innovation_z
Advanced:     hurst, return_skew_20, return_kurt_20, vol_regime
Sentiment:    sentiment (0.0 during training)
```

Features removed from earlier versions: `esn_signal` (cross-ticker comparability issues resolved by separate z-score normalisation but still excluded for training stability), `pe_ratio`, `forward_pe`, `pb_ratio` (near-zero SHAP importance), `above_sma20` (zero importance), `above_kalman_upper`, `below_kalman_lower` (redundant with `kalman_deviation`).

**Recency Weighting**

`np.linspace(0.5, 1.0, n_samples)` assigns linearly increasing weights to training samples, with the most recent data weighted at 1.0 and the oldest at 0.5. An additional 50% weight boost is applied to recent BUY label rows. This reflects the assumption that recent market behaviour is more predictive than behaviour from two years ago.

**Optuna Hyperparameter Tuning**

TPE (Tree-structured Parzen Estimator) sampler, hybrid objective function: `0.5 × Average Precision + 0.5 × F1`.

Pure AP optimisation tends to produce models that rank well but make poor binary decisions. Including F1 forces the model to also optimise the quality of its BUY vs SELL classifications at the optimal threshold.

`scale_pos_weight` is fixed to `counts[0] / counts[1]` and is not included in the Optuna search space. Including it caused Optuna to overfit the class weight to the training data, producing overconfident predictions on the minority class.

**Walk-Forward Cross-Validation**

Standard k-fold cross-validation is inappropriate for time series because it allows information from future periods to appear in the training set for earlier periods.

Walk-forward CV uses an expanding training window:

```
Window 1: Train [rows 0 to 599]    Test [rows 600 to 629]
Window 2: Train [rows 0 to 629]    Test [rows 630 to 659]
...
```

Parameters: `train_window=600`, `test_window=40`, `step=30`.

**Threshold Optimisation**

The default classification threshold of 0.5 assumes balanced classes. The optimal threshold is found by searching the range [0.45, 0.75] for the value that maximises F1 on the out-of-sample walk-forward predictions. This threshold is saved in the model pickle file and used at inference time.

Typical optimal threshold: 0.45 to 0.50. A threshold below 0.5 indicates the model is biased toward predicting SELL (expected given the 37% BUY label ratio) and needs a lower threshold to correctly identify enough BUY predictions.

**SHAP Explainability**

`shap.TreeExplainer` is run on a sample of 2,000 training rows after final model training. Top features by global SHAP importance from the most recent full training run:

| Feature | SHAP Importance |
|---|---|
| hurst | 0.2061 |
| qv_signature_ratio | 0.2030 |
| realized_vol_20d | 0.1932 |
| macd | 0.1888 |
| ou_theta | 0.1720 |
| vol_regime | 0.1625 |
| return_skew_20 | 0.1430 |
| rsi_14 | 0.1391 |

These features are economically sensible. The top three all measure volatility regime: Hurst captures trending vs mean-reverting, quadratic variation captures noise level, and realised vol captures absolute volatility. The model has learned that regime context is more predictive than any single price indicator.

**Adversarial Drift Detection**

A `RandomForestClassifier` is trained to distinguish between the last 200 rows of training data and the current live observation. AUC close to 0.5 indicates the live market looks similar to training data. AUC above 0.70 indicates the current regime significantly differs from anything seen during training. When the drift AUC exceeds 0.70, the signal output is changed to `WAIT/SELL` regardless of the XGBoost probability.

For very small sample sizes (fewer than 6 rows), the function falls back to Mahalanobis distance as a simpler drift measure.

**Model Artefacts**

- `4_signals/xgboost_global_model.pkl`: The trained model, scaler, feature column list, optimal threshold, and metrics. Loaded by `signal_engine.py` at inference time.
- `4_signals/esn_model.pkl`: The trained ESN model.

### `4_signals/signal_engine.py`

Orchestrates the full prediction pipeline for a single ticker on a single date. This is the primary interface used by the backtesting engine and the RL agent.

**`get_full_signals(df, ticker)`** processes a full OHLCV dataframe and returns a dataframe with one row per date containing all signal columns: `proba_buy`, `confidence`, `label`, `drift_auc`, `regime_warning`, `threshold_used`.

**`predict(df, ticker)`** runs prediction on the most recent row of a prepared feature dataframe. Returns the signal dictionary.

**`get_state(ticker, date)`** fetches fresh data, builds features, and returns a structured state vector for use as the RL agent's observation. This method was added specifically for the `TradingEnv` gymnasium environment in Section 5.

```python
{
    "signal": "BUY" / "SELL" / "WAIT",
    "state_vector": {
        "confidence":  float,   # XGBoost max probability
        "sentiment":   float,   # FinBERT score, 0.0 if unavailable
        "esn_zscore":  float,   # normalised ESN decision value
        "hurst":       float,   # current Hurst exponent
        "kalman_dist": float,   # Kalman deviation from smoothed price
        "vol_regime":  float,   # ATR ratio short/long
        "rsi":         float,   # RSI(14) normalised to [0, 1]
    }
}
```

**Sentiment overlay.** After XGBoost produces a BUY signal, the real-time FinBERT sentiment score is applied as a post-prediction adjustment:

- Sentiment below -0.3 with BUY signal: confidence multiplied by 0.75. If adjusted confidence falls below 0.5, label changed to WAIT/SELL.
- Sentiment above +0.3 with BUY signal: confidence multiplied by 1.15, capped at 0.99.
- Neutral sentiment: no adjustment.

This approach avoids contaminating the training data with sentiment values while still using sentiment information at inference time.

## Known Issues

**Walk-forward variance.** Walk-forward accuracy standard deviation remains high at approximately 0.13. This reflects genuine instability: the model performs well in some market regimes and poorly in others. Adding regime-conditional model selection (training separate models per regime) is the most likely path to reducing this variance. This is deferred to Section 6.

**Fundamental features near-zero importance.** `pe_ratio`, `forward_pe`, `pb_ratio`, `debt_equity`, `roe`, `beta` all show negligible SHAP importance. These features are static per training date and do not vary row by row, giving XGBoost little information to work with. They will be removed in a future revision.

**ESN cross-ticker comparability.** While z-score normalisation addresses the magnitude problem, each ticker's ESN is trained on its own price history with its own training split. The reservoir weights are random but fixed (seed=42), making the reservoir identical across tickers, but the readout layer (the only trained component) captures ticker-specific patterns. This is acceptable but means ESN signals are not fully interchangeable across tickers when used as features in the global model.
